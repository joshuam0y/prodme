prod.me — Supabase auth email templates
=======================================

Supabase does not read these files automatically on the hosted dashboard.
Copy the HTML from each file into the Dashboard:

  Dashboard → Authentication → Email Templates

1) Confirm signup
   - Subject: Confirm your prod.me account
   - Body: paste full contents of confirmation.html

2) Reset password
   - Subject: Reset your prod.me password
   - Body: paste full contents of recovery.html

3) Magic Link (if you enable passwordless email sign-in)
   - Subject: Your prod.me sign-in link
   - Body: paste full contents of magic_link.html

Optional: add a logo image
--------------------------
Upload a PNG (e.g. 240×64) to your site public/ folder, deploy, then in the
templates replace the text wordmark block with:

  <img src="https://YOUR_DOMAIN/logo.png" width="120" alt="prod.me"
       style="display:block;margin:0 auto 12px;border:0;" />

Google Fonts in email are often stripped; templates use system fonts
(Segoe UI, Roboto, Helvetica, Arial) for reliable rendering in Gmail/Outlook.

Variables must stay exactly as Supabase expects: {{ .ConfirmationURL }}, {{ .Email }}, etc.

Password reset / auth callback
-----------------------------
Under Authentication → URL configuration → Redirect URLs, allow your callback on every domain:
  https://YOUR_PRODUCTION_DOMAIN/auth/callback
  https://YOUR_PRODUCTION_DOMAIN/auth/callback?next=%2Fupdate-password
(local + preview URLs as needed)

If reset links redirect to Site URL with an error, the redirect URL is not allowlisted.
